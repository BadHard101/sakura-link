## О проекте

**SakuraLink** - клиент-серверное веб-приложение туристического справочника по Японии, ориентированное на русскоязычных пользователей. Проект решает проблему разрозненной и часто англоязычной информации о поездке в Японию: объединяет каталог городов и достопримечательностей, статьи, форум, пользовательские отзывы, рейтинги, избранное и real-time взаимодействие.

Приложение построено как клиент-серверная система: React SPA отвечает за интерфейс и пользовательские сценарии, Spring Boot реализует REST API, бизнес-логику, безопасность и доступ к данным, MySQL хранит пользователей, контент, оценки, отзывы и форумные обсуждения.

Основной фокус проекта - показать production-oriented full-stack навыки: проектирование REST API, работу с реляционной БД, JWT-аутентификацию, ролевую модель доступа, SPA-интерфейс, управление состоянием и интеграцию frontend/backend.

## Архитектура

Проект использует клиент-серверную архитектуру с разделением ответственности между frontend, backend и базой данных.

```text
User Browser
    |
    | HTTP / REST / JSON
    v
React SPA
    |
    | Axios + JWT Bearer Token
    v
Spring Boot REST API
    |
    | Spring Data JPA / Hibernate
    v
MySQL Database
```

### Frontend-слой

React-приложение отвечает за отображение интерфейса, маршрутизацию, работу с формами, хранение состояния и взаимодействие с сервером. Для API-запросов используется отдельный слой сервисов на Axios. При успешной авторизации JWT сохраняется на клиентской стороне и передается в заголовке `Authorization: Bearer <token>` при обращении к защищенным ресурсам.

Пример логической структуры frontend-части:

```text
src/
├── assets/       # изображения, иконки, статические ресурсы
├── components/   # переиспользуемые UI-компоненты
├── config/       # конфигурация приложения, API, store, routes
├── pages/        # страницы, связанные с маршрутами
├── redux/        # store, slices, actions, reducers
└── utils/        # вспомогательные функции
```

### Backend-слой

Spring Boot-приложение реализует REST API, бизнес-логику, аутентификацию, авторизацию и доступ к данным. Backend построен по слоеному принципу: контроллеры принимают HTTP-запросы, сервисы выполняют бизнес-логику, репозитории отвечают за работу с БД, а DTO отделяют внешние запросы и ответы от внутренних JPA-сущностей.

Пример логической структуры backend-части:

```text
src/main/java/.../
├── config/       # Spring Security, JWT, CORS, общие настройки
├── controller/   # REST-контроллеры
├── models/       # JPA-сущности
├── repository/   # Spring Data JPA репозитории
├── request/      # DTO для входящих запросов
├── response/     # DTO для ответов клиенту
└── service/      # бизнес-логика приложения
```

## Безопасность

В проекте реализована JWT-аутентификация на базе Spring Security.

Основные элементы безопасности:

- регистрация и авторизация через `/auth/signup` и `/auth/signin`;
- генерация JWT после успешного входа;
- передача JWT в заголовке `Authorization: Bearer <token>`;
- фильтрация входящих запросов через Spring Security;
- проверка подписи и срока действия токена;
- извлечение пользователя и ролей из токена;
- разграничение доступа по ролям;
- хэширование паролей через BCrypt;
- защита внутренних моделей через DTO-слой.

## Полный список возможностей

### Туристический справочник

- просмотр списка городов Японии;
- просмотр достопримечательностей и туристических объектов конкретного города;
- карточки мест с описанием, рейтингом, отзывами и медиа;
- добавление мест в избранное;
- лайки и дизлайки для оценки полезности контента;
- отзывы о местах с текстом, фотографиями и рейтингом от 1 до 5;
- ссылки на короткие видео, связанные с конкретными местами.

### Статьи и гайды

- публикация статей, обзоров и туристических гайдов;
- просмотр статей по категориям и отдельным страницам;
- комментарии к статьям, включая ответы друг на друга;
- голосование за полезные статьи;
- сохранение понравившихся материалов.

### Форум и Q&A

- создание тем на форуме;
- публикация сообщений внутри тем;
- иерархические ответы на сообщения;
- голосование за полезные ответы;
- отметка актуальных и популярных обсуждений;
- жалобы на неподходящий контент;
- модерация пользовательских публикаций.

### Профиль пользователя

- регистрация и авторизация;
- личный профиль пользователя;
- обновление данных профиля;
- загрузка аватара и фона профиля;
- просмотр избранных мест и материалов;
- участие в обсуждениях и публикация собственного контента.
